Summary Perubahan Konversi Cypress ke Playwright

Scope sumber Cypress yang dikonversi:
- cypress/e2e/01-v2_regression
- cypress/e2e/02-v2_test

Perubahan utama:
- Menambahkan spec Playwright baru untuk area auth, conversation, ticket, team, dan role validation.
- Melengkapi coverage file check-all/navigation agar mencakup register, ticket, broadcast, statistic, contact, setting, dan akses conversation tambahan.
- Memperbaiki support export Playwright agar page object yang dipakai test benar-benar tersedia dari playwright/support/pages/index.js.
- Memperbaiki helper auth Playwright supaya flow login invalid tidak selalu menunggu redirect sukses.
- Memperbaiki helper endpoint detection agar pencatatan request dilakukan dengan route tunggal dan file log disimpan via Node fs, bukan browser context.
- Memperbaiki file Playwright lama yang masih syntax error karena penutup blok belum lengkap.

Catatan penting hasil konversi:
- Tidak semua flow Cypress bisa dibuat runnable penuh tanpa risiko karena beberapa test bersifat mutasi data, bergantung pada email verification, onboarding document upload, atau account fixture spesifik environment.
- Untuk kasus seperti itu, test ditandai dengan test.fixme agar gap konversi tetap terdokumentasi jelas dan tidak menghasilkan false green atau test merah palsu.
- Team/member mutation sengaja tidak dibuat runnable penuh untuk menghindari perubahan data organisasi test secara tidak aman.
- Empty-state conversation saat ini runnable penuh hanya jika account dummy prod tersedia, sesuai data yang sudah ada di konfigurasi Playwright.

Verifikasi yang dilakukan:
- Syntax check seluruh file di playwright/tests/e2e dengan node --check: berhasil.
- Syntax check file support yang diubah: berhasil.
- Eksekusi playwright test penuh belum bisa dijalankan di environment ini karena package @playwright/test belum terpasang meskipun sudah tercantum di package.json.

Rekomendasi lanjut:
- Install dependency Playwright terlebih dulu, lalu jalankan playwright test list/smoke.
- Prioritaskan pelengkapan locator dan fixture untuk register validation, onboarding, dan team mutation supaya test.fixme bisa dikurangi.
