# AI Rules – Cypress Automation Project

## General Principles

- AI hanya boleh membaca dan menulis file di dalam folder project automation.
- Jangan mengakses folder sistem, user home, atau file di luar workspace.
- Jangan menjalankan command shell kecuali diminta secara eksplisit oleh user.

## Cypress Test Structure

Gunakan struktur berikut:

```
cypress/
  e2e/
  fixtures/
  support/
```

Semua test case harus berada di folder:

```
cypress/e2e
```

---

## Selector Rules

Selector harus menggunakan atribut stabil.

Prioritas selector:

1. `data-testid`
2. `data-cy`
3. `data-test`
4. id
5. class (hindari jika memungkinkan)

Contoh:

```javascript
cy.get("[data-testid=login-button]");
```

---

## Coding Style

Ikuti aturan berikut:

- Hindari penggunaan single quote
- Gunakan double quote untuk string
- Pisahkan selector dan action
- Jangan membuat command terlalu kompleks

Contoh yang benar:

```javascript
cy.get("[data-testid=email-input]");
cy.get("[data-testid=email-input]").type("test@mail.com");
```

---

## Test Case Pattern

Gunakan struktur berikut:

```javascript
describe("Feature Name", () => {
  beforeEach(() => {
    cy.visit("/");
  });

  it("should perform expected behavior", () => {});
});
```

---

## Error Handling

Setiap assertion harus jelas.

Gunakan:

```javascript
should();
```

Contoh:

```javascript
cy.url().should("include", "/dashboard");
```

---

## Automation Focus

Prioritaskan pembuatan test untuk:

- login flow
- chat message flow
- assignment flow
- inbound message
- outbound message
- error handling

---

## Refactoring Rules

Jika menemukan code yang:

- duplikat
- selector tidak stabil
- command terlalu panjang

AI harus menyarankan refactor.

---

## Security

AI tidak boleh:

- membaca file `.env`
- membaca credential
- membaca token API
- mengirim data project keluar workspace

---

## Goal

Tujuan AI dalam project ini:

- membantu membuat Cypress automation
- mempercepat pembuatan test
- meningkatkan reliability test
- mengurangi flaky test
