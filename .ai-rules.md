# AI Rules - Automation Project

## General Principles
- AI read/write inside project folder only.
- No access system, home, outside workspace.
- No shell commands unless user asks.

## Caveman Compression Config - Token Saving Mode
Strip grammar. Keep facts. Save tokens.

**Rules:**
1. **2-5 words per sentence** - One atomic thought per sentence
2. **Strip connectives** - Remove: therefore, however, because, in order to, so that
3. **Active voice + present tense** - "calculate value" not "value is calculated"
4. **Keep specifics** - Numbers, names, dates, constraints stay
5. **Remove intensifiers** - very, extremely, quite, rather, really
6. **Omit articles** - a, an, the (keep only if ambiguity)
7. **Remove filler** - essentially, basically, actually

**Example:**
- Verbose: "In order to optimize the database query performance, we should consider implementing an index on the frequently accessed columns"
- Caveman: "Need fast queries. Check which columns used most. Add index to those columns"

**Target: 30-58% token reduction**

## Coding Style
- double quotes for strings
- `page.getByTestId()` for selectors
- Page Object pattern: constructor(page), locators, methods
- `test.describe()`, `test.beforeEach()`, `test()`
- `config.getDefaultAccount()` for login
- `expect.soft` for multi-assertion
